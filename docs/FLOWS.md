# FLOWS

## Поток выезда мастера
Ниже описан целевой сквозной процесс работы с выездной заявкой — от создания до оплаты.

### 1) Создание заявки
Клиент создает заявку в приложении:
- выбирает категорию работ;
- указывает адрес и удобное время;
- добавляет описание проблемы, фото/видео при необходимости;
- подтверждает отправку заявки.

Результат: заявка получает первичный статус «Создана».

### 2) Рассылка мастерам
Система выполняет подбор и рассылку:
- отбор мастеров по геолокации, специализации и уровню;
- учет доступности и дисциплины мастера;
- отправка push/SMS/in-app уведомлений о новой заявке.

Результат: заявка переходит в статус «Разослана».

### 3) Принятие заявки
Мастер принимает заявку:
- изучает описание;
- подтверждает готовность выезда;
- фиксирует ориентировочное время прибытия.

Результат: заявка закрепляется за конкретным мастером, статус «Принята».

### 4) Выезд
Мастер начинает выезд к клиенту:
- переводит заявку в статус «В пути»;
- клиент видит обновление статуса и ETA;
- при необходимости мастер связывается с клиентом для уточнений.

Результат: активный этап исполнения с контролем времени.

### 5) Осмотр
На объекте мастер проводит осмотр:
- диагностика фактического объема работ;
- фотофиксация состояния;
- согласование технических вводных с клиентом.

Результат: статус «Осмотр выполнен», входные данные для сметы готовы.

### 6) Смета
Мастер формирует смету (через Сметмастер):
- перечень работ и материалов;
- стоимость по позициям;
- сроки выполнения.

Клиент рассматривает и согласует смету.

Результат: после подтверждения клиентом статус «Смета согласована».

### 7) Выполнение
Мастер приступает к работам:
- выполнение задач по согласованной смете;
- фиксация промежуточных этапов при необходимости;
- коммуникация с клиентом по отклонениям и дополнительным работам.

Результат: статус «Выполнение» с операционным контролем.

### 8) Завершение
После окончания работ:
- мастер отмечает заказ как выполненный;
- клиент проверяет результат;
- при отсутствии замечаний подтверждает завершение.

Результат: статус «Завершен».

### 9) Оплата
Финальный расчет по заказу:
- клиент производит оплату;
- система фиксирует транзакцию;
- автоматически рассчитывается комиссия платформы 5%;
- остаток зачисляется мастеру с учетом финансовых правил.

Результат: заказ закрыт, финансовый цикл завершен.
